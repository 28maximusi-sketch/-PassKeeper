PassKeeper – Локальный менеджер паролей на 7 языках
Безопасное хранилище паролей в зашифрованном файле с мастер-паролем.
Поддержка командной строки: добавление, получение, список, удаление, смена мастер-ключа.
Реализован на 7 языках с единым интерфейсом и продвинутыми возможностями.

🚀 Возможности
Шифрование данных (AES-256-GCM) на основе мастер-пароля.

Ключ формируется с помощью PBKDF2 (100 000 итераций, SHA-256).

Хранилище – один JSON-файл в домашней директории (например, ~/.passkeeper.db).

Команды:

master <пароль> – установить или сменить мастер-пароль.

add <имя> <логин> <пароль> [URL] – добавить запись.

get <имя> – показать логин и пароль для указанной записи.

list – показать все имена записей.

del <имя> – удалить запись.

Дополнительные фичи:

Автоматическое создание хранилища при первом запуске.

Проверка мастер-пароля при каждой операции.

Генерация случайной соли для PBKDF2.

Поддержка Unicode в данных.

Цветной вывод (где поддерживается).

📖 Использование
Синтаксис (единый для всех версий):

bash
<команда> <субкоманда> [аргументы]
Субкоманды
Команда	Описание
master <пароль>	Установить/сменить мастер-пароль (если файл уже существует, требуется старый пароль)
add <имя> <логин> <пароль> [URL]	Добавить запись с указанными данными
get <имя>	Показать логин и пароль для записи
list	Показать список всех имён записей
del <имя>	Удалить запись
Примеры
bash
# Установить мастер-пароль
python passkeeper.py master MyStrongMasterPassword

# Добавить запись
python passkeeper.py add github mylogin mypassword https://github.com

# Получить пароль
python passkeeper.py get github

# Список всех записей
python passkeeper.py list

# Удалить запись
python passkeeper.py del github
🛠 Установка и запуск
Python
bash
python passkeeper.py <субкоманда> ...
Требуется Python 3.6+ и библиотека cryptography (установить через pip install cryptography).

Go
bash
go build passkeeper.go
./passkeeper <субкоманда> ...
JavaScript (Node.js)
bash
node passkeeper.js <субкоманда> ...
Требуется Node.js и пакет crypto-js (установить через npm install crypto-js).

C++
bash
g++ -std=c++11 passkeeper.cpp -lssl -lcrypto -o passkeeper
./passkeeper <субкоманда> ...
Требуется OpenSSL.

C#
bash
csc passkeeper.cs
mono passkeeper.exe <субкоманда> ...   # или dotnet run
Требуется .NET Framework с System.Security.Cryptography.

Java
bash
javac passkeeper.java
java passkeeper <субкоманда> ...
Ruby
bash
ruby passkeeper.rb <субкоманда> ...
Требуется гем openssl (обычно встроен).

🧠 Архитектура
Хранилище: файл .passkeeper.db в домашней директории пользователя.

Формат: JSON с полями salt (hex), data (зашифрованный текст в base64).

Шифрование:

Ключ: PBKDF2 (мастер-пароль + соль, 100 000 итераций, SHA-256, длина 32 байта).

Шифр: AES-256-GCM (или AES-CBC с HMAC для простоты, но предпочтительнее GCM).

Данные: JSON-объект с записями (ключ – имя записи, значение – объект с login, password, url).

Команды: парсер аргументов, проверка мастер-пароля при каждом доступе к данным.

✨ Дополнительные фичи
Автоматическое создание файла хранилища с новой солью при первом master.

Проверка корректности мастер-пароля перед операциями (кроме master).

Генерация случайной соли (16 байт) с помощью криптостойкого ГСЧ.

Безопасное удаление старых данных при смене пароля (перезапись файла).

Цветной вывод для успешных/ошибочных операций.

Поддержка пробелов в аргументах (используйте кавычки).
